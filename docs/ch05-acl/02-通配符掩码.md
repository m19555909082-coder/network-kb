### 📄 Slide 11：应用ACL  Applying ACL

<div class="bilingual-cn" markdown>

**🇨🇳 中文**

如果只是定义了ACL，它还不会起到任何作用，必须把ACL应用到一个接口上才能起作用。
in：表示在数据包进入此接口时使用ACL进行过滤。
out：表示在数据包离开此接口时使用ACL进行过滤。
通常，使用出站接口检查的数据包数量较少，效率要高一些。

</div>

<div class="bilingual-en" markdown>

**🇬🇧 English**

应用ACL  Applying ACL
If the ACL is just defined, it doesn't do anything yet; it must be applied to an interface to work.
应用ACL：Applying ACL
Router(config)# interface 接口号
Router(config-if)# ip access-group 表号 [in | out]
In：Indicates that ACLs are used to filter packets as they enter this interface.
out:  Indicates that a packet is filtered using an ACL when it leaves this interface.
In general, the number of packets examined using the outbound interface is smaller and more efficient.

</div>
**💻 配置命令：**

```cisco
If the ACL is just defined, it doesn't do anything yet; it must be applied to an interface to work.
Router(config)# interface 接口号
Router(config-if)# ip access-group 表号 [in | out]
In general, the number of packets examined using the outbound interface is smaller and more efficient.
```

### 📄 Slide 13：通配符掩码 Wildcard Mask

<div class="bilingual-cn" markdown>

**🇨🇳 中文**

在ACL语句中，当使用地址作为条件时，它的一般格式为：地址 通配符掩码。
通配符掩码决定了地址中的哪些位需要精确匹配，哪些位不需要匹配。
通配符掩码是一个32位数，采用点分十进制方式书写。匹配时，“0”表示检查的位，“1”表示不检查的位。

</div>

<div class="bilingual-en" markdown>

**🇬🇧 English**

通配符掩码 Wildcard Mask
In an ACL statement, when an address is used as a condition, its general format is: Address wildcard mask.
The wildcard mask determines which bits in the address need to be matched exactly and which bits do not.
The wildcard mask is a 32 bits number written in dotted decimal. In case of a match, "0" indicates the checked bit and "1" indicates the bit that is not checked.

</div>

表示检查前16位，忽略后16位，所以这个条件表示的地址是 192.168.*.*。

<div class="bilingual-en" markdown>

通配符掩码 Wildcard Mask
如：192.168.1.1  0.0.255.255
For example: 192.168.1.1  0.0.255.255
表示检查前16位，忽略后16位，所以这个条件表示的地址是 192.168.*.*。
The first 16 bits are checked and the last 16 bits are ignored, so this condition represents the address 192.168.*.*.

</div>
