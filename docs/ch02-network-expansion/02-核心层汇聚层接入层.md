### 📄 Slide 8：层次化网络规划设计

**🇨🇳 中文**

层次化网络规划设计
核心层（网络的高速数据转发）：对整个网络的连通起到至关重要的作用。核心层应该具有如下几个特性：可靠性、高效性、冗余性、容错性、可管理性、适应性、低延时性等。在核心层中，应该采用高带宽的千兆以上交换机。因为核心层是网络的枢纽中心，重要性突出。核心层设备采用双机冗余热备份是非常必要的，也可以使用负载均衡功能，来改善网络性能。

</div>

<div class="bilingual-en" markdown>

**🇬🇧 English**

Hierarchical network planning and design
The core layer (the high-speed data forwarding of the network) : it is crucial for the connectivity of the entire network. The core layer should have the following characteristics: reliability, efficiency, redundancy, fault tolerance, manageability, adaptability, low latency, etc. In the core layer, high bandwidth gigabit + switches should be employed. Because the core layer is the hub center of the network, its importance is prominent. It is very necessary for core layer devices to adopt dual redundant hot backup, and load balancing function can also be used to improve network performance.

</div>

### 📄 Slide 9：层次化网络规划设计

<div class="bilingual-cn" markdown>

**🇨🇳 中文**

层次化网络规划设计
汇聚层（路由聚合及流量收敛）：是在工作站接入核心层前先做汇聚，以减轻核心层设备的负荷。汇聚层具有实施策略、安全、工作组接入、虚拟局域网（VLAN）之间的路由、源地址或目的地址过滤等多种功能。在汇聚层中，应该选用支持三层交换技术和VLAN的交换机，以达到网络隔离和分段的目的

</div>

<div class="bilingual-en" markdown>

**🇬🇧 English**

Hierarchical network planning and design
Aggregation layer (route aggregation and traffic convergence) : it does aggregation before workstations access the core layer to reduce the load of core layer devices. The aggregation layer has many functions such as enforcing policy, security, workgroup access, routing between virtual Local Area Networks (VLans), source or destination address filtering, and so on. In the aggregation layer, the switch supporting layer 3 switching technology and VLAN should be selected to achieve the purpose of network isolation and segmentation

</div>

层次化网络规划设计
接入层 （工作组接入及访问控制）：向本地网段提供工作站接入。在接入层中，减少同一网段的工作站数量，能够向工作组提供高速带宽。接入层可以选择不支持VLAN和三层交换技术的普通交换机。

</div>

<div class="bilingual-en" markdown>

**🇬🇧 English**

Hierarchical network planning and design
Access layer (Workgroup access and Access Control) : Provides workstation access to local network segments. In the access layer, reducing the number of workstations in the same network segment can provide high-speed bandwidth to workgroups. The access layer can choose normal switches that do not support VLAN and Layer 3 switching technologies.

</div>

### 📄 Slide 11：层次化网络规划设计

**🇨🇳 中文**

层次化网络规划设计
流量收敛举例：假设你有10台服务器，每台服务器通过10GE的接口连接到一个接入交换机，那我们一共就有100G（10×10G=100G）的南向带宽。假设这台交换机还有2个40GE的接口可以用于接入到更高一层的汇聚交换机，那我们一共就有80G（2×40G=80G）的北向带宽。此时，我们得到的收敛比则是1.25：1（100G÷80G=1.25）。

<div class="bilingual-en" markdown>

**🇬🇧 English**

Hierarchical network planning and design
流量收敛举例：假设你有10台服务器，每台服务器通过10GE的接口连接到一个接入交换机，那我们一共就有100G（10×10G=100G）的南向带宽。假设这台交换机还有2个40GE的接口可以用于接入到更高一层的汇聚交换机，那我们一共就有80G（2×40G=80G）的北向带宽。此时，我们得到的收敛比则是1.25：1（100G÷80G=1.25）。
Convergence Example: If you have 10 servers, each connected to an access switch through a 10GE interface, then we have 100G (10 x 10G=100G) of southbound bandwidth. Assuming that this switch also has two 40GE interfaces that can be used to connect to the higher-layer aggregation switch, we have a total of 80G (2 x 40G=80G) of northbound bandwidth. In this case, we get a convergence ratio of 1.25:1 (100G÷80G=1.25).

### 📄 Slide 12：层次化网络规划设计

<div class="bilingual-cn" markdown>

**🇨🇳 中文**

层次化网络规划设计
需要说明的是，造成网络流量收敛的原因并不总是上述描述的这个例子，不过总的来说，我们可以将流量收敛的原因分为两类：
交换机不支持线速转发，在交换机内部可能形成流量收敛；
网络架构设计的原因，无论交换机是否线速，转发报文时也会存在流量收敛。
线速转发：端口在满负载的情况下，对帧进行无差错的转发称为线速转发

</div>

<div class="bilingual-en" markdown>

**🇬🇧 English**

Hierarchical network planning and design
It should be noted that the causes of convergence of network traffic are not always the examples described above, but in general, we can classify the causes of convergence of traffic into two categories:
The switch does not support wire-speed forwarding, and traffic convergence may occur inside the switch.
Because of the network architecture design, whether the switch is wire-speed or not, there will also be traffic convergence when forwarding packets.
Wire-speed forwarding: When a port is fully loaded, error-free forwarding of frames is called wire-speed forwarding
